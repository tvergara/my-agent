# Citation Formatting Audit: b044e3c3

Paper: A Unified SPD Token Transformer Framework for EEG Classification: Systematic Comparison of Geometric Embeddings
ID: b044e3c3-4a8e-4a74-a3b8-13584deba079

## Observations

The following issues were identified in the citation formatting of the paper, specifically within `REFERENCES.bib`:

### 1. Unprotected Acronyms and Proper Names in Titles
In ICML's bibliography style, titles are typically converted to sentence case. Acronyms and proper names must be protected with curly braces `{...}` to prevent them from being incorrectly lowercased (e.g., "BCI" becoming "bci").

The following entries in `REFERENCES.bib` are missing this protection:
- `barachant2012classification`: "BCI" -> `{BCI}`
- `arsigny2006log`: "Log-Euclidean" -> `{Log-Euclidean}`
- `lawhern2018eegnet`: "EEGNet", "EEG-based", "BCI"
- `schirrmeister2017deep`: "EEG"
- `nikolopoulos2017mamem`: "MAMEM"
- `ingolfsson2020eegtcnet`: "EEG-TCNet"
- `ingolfsson2021fbconet`: "FBCNet"
- `song2022eegconformer`: "EEG" (in "EEG conformer")
- `zhang2023lmda`: "LMDA-Net"
- `chen2024mind`: "EEG"
- `chen2024necomimi`: "Necomimi", "EEG-informed"

### 2. Redundant Information in Booktitles
Several conference entries include the year in the `booktitle` field despite it already being in the `year` field. This can lead to redundant output like "2020 2020 IEEE International Conference on...".
- `ingolfsson2020eegtcnet`: `booktitle={2020 IEEE International Conference on Systems, Man, and Cybernetics (SMC)}`
- `ingolfsson2021fbconet`: `booktitle={2021 IEEE International Conference on Systems, Man, and Cybernetics (SMC)}`

### 3. Inconsistent arXiv Formatting
The paper uses `arXiv preprint arXiv:XXXX.XXXXX` in the `journal` field for some entries (e.g., `bronstein2021geometric`, `chen2024mind`), which is a common but older convention. Modern BibTeX styles often prefer using the `eprint` and `archivePrefix` fields.
