# Bibliography and Citation Audit: Transport Clustering (d50ca57f)

## Summary
I have performed a systematic audit of the bibliography and citation formatting for the submission "Transport Clustering: Solving Low-Rank Optimal Transport via Clustering". While the paper is technically sound, there are several significant formatting issues in both the LaTeX source and the BibTeX database that violate ICML's expected standards for reference integrity and professional presentation.

## Detailed Findings

### 1. Inconsistent Citation Commands
The paper inconsistently mixes `\cite`, `\citep`, and `\citet`. In an author-year style (standard for ICML), these commands have distinct semantic roles. 
- **Inconsistency**: I found 87 uses of `\cite`, 92 uses of `\citep`, and only 1 use of `\citet`.
- **Misuse**: Multiple instances were identified where `\cite` is used in a narrative context where `\citet` (author as subject) is required.
    - Example (Line 429): `Following this observation, \cite{scetbon2022lowrank}` renders as "Following this observation, (Scetbon & Cuturi, 2022)" instead of "Following this observation, Scetbon & Cuturi (2022)".
    - Example (Line 499): `\cite{zhuang2023statistically}. In particular...`

### 2. Bibliography Capitalization Protection
The `icml2026.bst` style enforces sentence case for titles. Proper nouns and acronyms must be protected with curly braces `{}` to avoid being lowercased.
- **Missing Protection**: Key proper nouns are unprotected in many entries, which will lead to incorrect rendering (e.g., "monge", "kantorovich", "gromov", "wasserstein").
    - `title = {The Monge Gap: ...}` -> "The monge gap"
    - `title = {Learning cell fate landscapes ... using Fused Gromov-Wasserstein}` -> "... using fused gromov-wasserstein"
- **Erratic Protection**: Some entries use non-standard per-letter protection (e.g., `title={Low-rank {O}ptimal {T}ransport...}`) while others use double braces (e.g., `title={{The Hungarian Method...}}`), leading to an inconsistent visual style in the final bibliography.

### 3. Metadata and Entry Errors
- **Duplicate Fields**: Entry `scetbon2022lowrank` contains two `year = {2022}` fields.
- **Unusual Formatting**: Entries like `scetbon2022lowrank` and `FRLC` have a newline immediately after the opening brace, which while technically valid, can sometimes cause issues with simpler BibTeX parsers or scripts.
- **Unused References**: The `references.bib` file contains 234 unused entries. While not a direct error, it is best practice to provide a focused bibliography.

## Recommendations
- Standardize on `\citet` for narrative citations and `\citep` for parenthetical citations.
- Systematically add `{}` protection to all proper nouns and acronyms in `references.bib`.
- Remove duplicate fields and unify the bracing style for title protection.
