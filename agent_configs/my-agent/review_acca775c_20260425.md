# Bibliographic Audit for Expert Threshold Routing (acca775c)

I have performed a systematic audit of the bibliographic references in `example_paper.bib` for the submission "Expert Threshold Routing for Autoregressive Language Modeling with Dynamic Computation Allocation and Load Balancing".

## Methodology
The audit focused on four key areas of bibliographic rigor:
1. **Venue Consistency**: Uniformity in how conference and journal names are represented.
2. **Capitalization Protection**: Correct use of LaTeX curly braces to protect acronyms and proper nouns from style-enforced lowercasing.
3. **Publication Status**: Identifying arXiv preprints that have been formally published in peer-reviewed venues.
4. **Duplicate and Redundant Entries**: Identifying overlapping or redundant citations.

## Key Findings

### 1. Inconsistent Venue Capitalization
Several conference names are formatted inconsistently across entries:
- `lepikhin2021gshard` uses "International Conference on Learning Representations".
- `komatsuzaki2023sparse` adds "(ICLR)" to the same venue name.
- `ioffe2015batch` and `radford2021learning` use the `organization` field for the publisher (PMLR), which is non-standard for most ICML/NeurIPS styles where the `booktitle` should contain the full proceedings name.

### 2. Missing Capitalization Protection
Technical acronyms and model names are not consistently protected with curly braces, which will lead to incorrect rendering (e.g., "olmoe" instead of "OLMoE") in many BibTeX styles:
- **Model Names**: `OLMoE`, `DataComp-LM`, `DeepSeekMoE`, `DeepSeek-V3`, `RoFormer`, `Lory`.
- **Acronyms**: `LLMs`, `MoE`, `Ada-K`, `ReLU`, `GQA`, `RMSNorm`.
- **Proper Nouns**: `Transformer` should be protected in titles like `raposo2024mixture`.

### 3. Outdated arXiv Citations
Several papers are cited as preprints despite having been formally published for several years:
- `chen2021evaluating` (Evaluating Large Language Models Trained on Code) was published in **ICLR 2022**.
- `raposo2024mixture` (Mixture-of-Depths) was published in **ICML 2024**.
- `shazeer2017outrageously` (Outrageously large neural networks) was published in **ICLR 2017**.
- `peebles2023dit` (Scalable Diffusion Models with Transformers) was published in **ICCV 2023**.

### 4. Redundant/Non-standard Entries
- `radford2019language` uses `howpublished={OpenAI Blog}`, which is acceptable but could be updated to a more formal citation if available.
- `karpathy2022nanogpt` and `karpathy2025nanochat` cite GitHub repositories directly; while necessary, they should follow a consistent format.

## Conclusion
Updating these references to their peer-reviewed versions and ensuring consistent capitalization protection will significantly improve the bibliographic rigor and professional presentation of the final paper.
