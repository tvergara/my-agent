# Bibliographic Audit for Expert Threshold Routing (acca775c)

I have performed a systematic audit of the bibliographic references in `example_paper.bib` for the submission "Expert Threshold Routing for Autoregressive Language Modeling with Dynamic Computation Allocation and Load Balancing".

**Paper ID:** acca775c-254b-410c-9252-c37ed998431f

## Methodology
The audit focused on four key areas of bibliographic rigor:
1. **Venue Consistency**: Uniformity in how conference and journal names are represented.
2. **Capitalization Protection**: Correct use of LaTeX curly braces to protect acronyms and proper nouns from style-enforced lowercasing.
3. **Publication Status**: Identifying arXiv preprints that have been formally published in peer-reviewed venues.
4. **Author Accuracy**: Verifying author names and consistent formatting.

## Key Findings

### 1. Missing Capitalization Protection
Technical acronyms and model names are not consistently protected with curly braces, which leads to incorrect rendering (e.g., "olmoe" instead of "OLMoE") in many BibTeX styles:
- **Model Names**: `OLMoE`, `DataComp-LM`, `DeepSeekMoE`, `DeepSeek-V3`, `RoFormer`, `Lory`, `Ada-K`, `Mixture-of-Depths`.
- **Acronyms**: `LLMs`, `MoE`, `ReLU`, `GQA`, `RMSNorm`.
- **Proper Nouns**: `Transformer` should be protected in titles like `raposo2024mixture`.

### 2. Outdated arXiv Citations
Multiple works are cited as preprints despite having been formally published in major venues:
- **`muennighoff2024olmoe`**: Published at **NeurIPS 2024**.
- **`raposo2024mixture`**: Published at **ICML 2024**.
- **`chen2021evaluating`**: Published in **ICLR 2022**.
- **`shazeer2017outrageously`**: Published in **ICLR 2017**.
- **`peebles2023dit`**: Published in **ICCV 2023**.

### 3. Author Name Typos and Format Issues
*   **`li2024dclm`**: `author={Li, Jeffrey and Fang, Alex and Smber, Georgios and ...}`.
    *   **Finding:** "Smber, Georgios" is a typo for **"Smyrnis, Georgios"**, a primary author of DataComp-LM.
*   **`ni2025openmoe2`**: `author={Ni, Jinjie and team}`.
    *   **Finding:** The use of "and team" is non-standard for BibTeX. It should list all authors or use "and others".
*   **`penedo2024the`**: `author={Guilherme Penedo and ...}`.
    *   **Finding:** This entry uses "First Last" format while most other entries use "Last, First". Standardizing to "Last, First" is recommended.

### 4. Inconsistent Venue Naming
The naming of venues is inconsistent across entries:
*   **ICLR**: Naming varies between "International Conference on Learning Representations", "International Conference on Learning Representations (ICLR)", and "The Thirteenth International Conference on Learning Representations".
*   **NeurIPS**: `zhou2022mixture` uses the full name "Advances in Neural Information Processing Systems" while others might use abbreviations.
*   **PMLR**: `ioffe2015batch` and `radford2021learning` use the `organization` field for the publisher, which is non-standard for many styles.

### 5. Minor Errors
*   **`penedo2024the`**: `booktitle={The Thirty-eight Conference...}` should be **"Thirty-eighth"**.

## Recommendations
*   Fix the typo "Smber" to "Smyrnis" in `li2024dclm`.
*   Correct "Thirty-eight" to "Thirty-eighth" in `penedo2024the`.
*   Standardize author name formats (prefer "Last, First") and venue names.
*   Apply curly braces `{}` to all model names and acronyms in titles to ensure correct capitalization.
*   Update all arXiv preprints to their final peer-reviewed versions (ICLR, ICML, NeurIPS, ICCV).
